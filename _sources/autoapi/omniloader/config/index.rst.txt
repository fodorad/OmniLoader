omniloader.config
=================

.. py:module:: omniloader.config

.. autoapi-nested-parse::

   Configuration and reproducibility helpers.

   :class:`OmniConfig` bundles the knobs a training run needs — seed, batch size,
   worker count, the mixing strategy and the normalization/augmentation pipeline —
   and loads from a dict, a JSON file or a YAML file so experiments stay
   declarative. :func:`seed_everything` seeds Python, NumPy and PyTorch in one call.

.. py:class:: OmniConfig

   Declarative configuration for an OmniLoader training run.

   :param seed: Global RNG seed.
   :param batch_size: Batch size for the ``DataLoader``.
   :param num_workers: Number of worker processes for the ``DataLoader``.
   :param pad_features: Whether the unifier pads/crops sequence values.
   :param strategy: Name of the mixing strategy (a key of :data:`STRATEGIES`).
   :param strategy_kwargs: Extra keyword arguments forwarded to the strategy
                           (e.g. ``temperature``, ``weights``, ``samples_per_dataset``).
   :param transforms: Ordered list of transform config dicts (each with a ``name``
                      key) applied during training. Stats-based normalizers may reference a
                      saved stats file via ``stats_path`` (or ``union_stats_path``) instead of
                      inlining ``stats``. See :func:`~omniloader.transforms.build_transform`.
   :param eval_transforms: Transforms applied during evaluation. When ``None``,
                           only the non-augmenting (train-agnostic) transforms in
                           :attr:`transforms`, such as ``normalize``, take effect.
   :param datasets: Declarative dataset entries (adapter + args + schema) built by
                    :meth:`build_datasets`. See :mod:`omniloader.data.factory`.
   :param subsample: Optional per-dataset subsampling entries (one per dataset, or
                     ``None`` to skip a dataset), each a dict for :class:`SubsampleConfig`.
                     Built by :meth:`build_subsample` and fed to :meth:`build_strategy`.
   :param collate: Collate function name — one of :data:`COLLATES` (``"unified"``,
                   ``"dynamic"`` or ``"mixup"``). See :meth:`build_collate`. ``"dynamic"``
                   only reduces padding with :attr:`pad_features` set to ``False`` (under a
                   fixed ``time_dim`` it behaves like ``"unified"``).
   :param collate_kwargs: Extra keyword arguments for the collator. For ``"dynamic"``
                          an optional ``keys`` list; for ``"mixup"`` a ``base`` collate name plus
                          ``alpha``/``mode``/``p``.
   :param bucketing: Optional length-bucketing entry with a ``key`` (sequence name to
                     bucket by) and optional ``batch_size``/``bucket_multiplier``/
                     ``drop_last``/``shuffle``. When set, :meth:`build_dataloader` batches
                     with :class:`~omniloader.sampling.bucketing.LengthBucketBatchSampler`.
                     Only meaningful with ``pad_features=False`` (a no-op under a fixed
                     ``time_dim``).
   :param num_replicas: Distributed replica count for the sampler. ``None`` infers
                        it from ``torch.distributed``.
   :param rank: This process's distributed rank. ``None`` infers it.
   :param drop_last: Drop the uneven tail when sharding across replicas.
   :param pin_memory: Copy tensors into pinned memory before returning them (faster
                      host-to-GPU transfer). Forwarded to the ``DataLoader``.
   :param persistent_workers: Keep worker processes alive between epochs. Only takes
                              effect when :attr:`num_workers` > 0.
   :param prefetch_factor: Batches prefetched per worker. Only applied when
                           :attr:`num_workers` > 0; ``None`` uses the PyTorch default.

   .. py:method:: build_loader(datasets: collections.abc.Sequence[omniloader.loader.SizedDataset] | None = None, schemas: collections.abc.Sequence[omniloader.schema.spec.DatasetSchema] | None = None, training: bool = True) -> omniloader.loader.OmniLoader

      Build a fully configured :class:`~omniloader.loader.OmniLoader`.

      :param datasets: Source datasets. When ``None``, they are built from
                       :attr:`datasets` via :meth:`build_datasets`.
      :param schemas: Per-dataset schemas. Required when ``datasets`` is given;
                      ignored (rebuilt) when ``datasets`` is ``None``.
      :param training: Whether augmentations are active (``False`` for val/test).

      :returns: The loader with its transform pipeline wired in.

      :raises ValueError: If ``datasets`` is given without ``schemas``.



   .. py:method:: build_dataloader(datasets: collections.abc.Sequence[omniloader.loader.SizedDataset] | None = None, schemas: collections.abc.Sequence[omniloader.schema.spec.DatasetSchema] | None = None, training: bool = True, epoch: int = 0) -> torch.utils.data.DataLoader

      Assemble a ready-to-iterate ``DataLoader`` entirely from this config.

      Wires together the loader, mixing strategy, distributed sampler, collate
      function and (optionally) length bucketing. Training loaders draw indices
      through the configured :class:`~omniloader.sampling.strategies.MixingStrategy`;
      evaluation loaders (``training=False``) iterate sequentially for a stable,
      full-coverage pass.

      :param datasets: Source datasets, or ``None`` to build them from
                       :attr:`datasets`.
      :param schemas: Per-dataset schemas (required with ``datasets``).
      :param training: Whether this is the training split.
      :param epoch: Initial epoch for the sampler and per-sample augmentation.

      :returns: A configured :class:`torch.utils.data.DataLoader`.
