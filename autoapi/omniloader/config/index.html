omniloader.config¶

Configuration and reproducibility helpers.

OmniConfig bundles the knobs a training run needs — seed, batch size, worker count, the mixing strategy and the normalization/augmentation pipeline — and loads from a dict, a JSON file or a YAML file so experiments stay declarative. seed_everything() seeds Python, NumPy and PyTorch in one call.

class omniloader.config.OmniConfig[source]¶

Declarative configuration for an OmniLoader training run.

Parameters:

  • seed – Global RNG seed.

  • batch_size – Batch size for the DataLoader.

  • num_workers – Number of worker processes for the DataLoader.

  • pad_features – Whether the unifier pads/crops sequence values.

  • strategy – Name of the mixing strategy (a key of STRATEGIES).

  • strategy_kwargs – Extra keyword arguments forwarded to the strategy (e.g. temperature, weights, samples_per_dataset).

  • transforms – Ordered list of transform config dicts (each with a name key) applied during training. Stats-based normalizers may reference a saved stats file via stats_path (or union_stats_path) instead of inlining stats. See build_transform().

  • eval_transforms – Transforms applied during evaluation. When None, only the non-augmenting (train-agnostic) transforms in transforms, such as normalize, take effect.

  • datasets – Declarative dataset entries (adapter + args + schema) built by build_datasets(). See omniloader.data.factory.

  • subsample – Optional per-dataset subsampling entries (one per dataset, or None to skip a dataset), each a dict for SubsampleConfig. Built by build_subsample() and fed to build_strategy().

  • collate – Collate function name — one of COLLATES ("unified", "dynamic" or "mixup"). See build_collate(). "dynamic" only reduces padding with pad_features set to False (under a fixed time_dim it behaves like "unified").

  • collate_kwargs – Extra keyword arguments for the collator. For "dynamic" an optional keys list; for "mixup" a base collate name plus alpha/mode/p.

  • bucketing – Optional length-bucketing entry with a key (sequence name to bucket by) and optional batch_size/bucket_multiplier/ drop_last/shuffle. When set, build_dataloader() batches with LengthBucketBatchSampler. Only meaningful with pad_features=False (a no-op under a fixed time_dim).

  • num_replicas – Distributed replica count for the sampler. None infers it from torch.distributed.

  • rank – This process’s distributed rank. None infers it.

  • drop_last – Drop the uneven tail when sharding across replicas.

  • pin_memory – Copy tensors into pinned memory before returning them (faster host-to-GPU transfer). Forwarded to the DataLoader.

  • persistent_workers – Keep worker processes alive between epochs. Only takes effect when num_workers > 0.

  • prefetch_factor – Batches prefetched per worker. Only applied when num_workers > 0; None uses the PyTorch default.

build_dataloader(datasets: collections.abc.Sequence[omniloader.loader.SizedDataset] | None = None, schemas: collections.abc.Sequence[omniloader.schema.spec.DatasetSchema] | None = None, training: bool = True, epoch: int = 0) → torch.utils.data.DataLoader[source]¶

Assemble a ready-to-iterate DataLoader entirely from this config.

Wires together the loader, mixing strategy, distributed sampler, collate function and (optionally) length bucketing. Training loaders draw indices through the configured MixingStrategy; evaluation loaders (training=False) iterate sequentially for a stable, full-coverage pass.

Parameters:

  • datasets – Source datasets, or None to build them from datasets.

  • schemas – Per-dataset schemas (required with datasets).

  • training – Whether this is the training split.

  • epoch – Initial epoch for the sampler and per-sample augmentation.

Returns:

A configured torch.utils.data.DataLoader.
