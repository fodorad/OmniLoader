omniloader.introspection¶

Introspection helpers: describe a dataset mix and validate it against specs.

describe() summarises a collection of datasets — the union schema, which dataset provides which key (the coverage matrix), the fraction of valid (non- placeholder) positions per key, and class histograms for integer targets. validate() dry-runs each dataset, checking that the raw samples it emits match the shape and rank their TensorSpec declares — catching schema/data mismatches before training starts.

omniloader.introspection.validate(datasets: collections.abc.Sequence[omniloader.loader.SizedDataset], schemas: collections.abc.Sequence[omniloader.schema.spec.DatasetSchema], num_samples: int = 4, strict: bool = False) → list[str][source]¶

Check that each dataset’s raw samples match its declared specs.

For every declared key it verifies the value’s rank and trailing feature size against the TensorSpec (the sequence length is free — unification pads it). Dtype is not checked because unification casts it.

Parameters:

  • datasets – The source datasets.

  • schemas – One schema per dataset.

  • num_samples – Number of samples to probe per dataset.

  • strict – When True, raise ValueError if any issue is found.

Returns:

A list of human-readable issue strings (empty when everything matches).

Raises:

ValueError – If strict and issues were found.
